Cafeteria Line Manager — Commands Explanation

HELP Shows the list of all available commands and their descriptions.
Example: HELP

ARRIVE Adds a person to the end of the queue and records their arrival
time. Example: ARRIVE Aliya Complexity: O(1)

VIP_ARRIVE Adds a person to the front of the queue (VIP). Example:
VIP_ARRIVE Amir Complexity: O(1)

SERVE Serves the first person in the queue and removes them. The program
calculates the waiting time: currentTime − arrivalTime. Example: SERVE
If the queue is empty: “No one to serve.” Complexity: O(1)

LEAVE Removes a person from the queue if they decide to leave. Example:
LEAVE Dana If the person is not found: “Not found” Complexity: O(n)
because the queue must be searched.

PEEK Shows who will be served next without removing them from the queue.
Example: PEEK If empty: “Line is empty.” Complexity: O(1)

SIZE Shows how many people are currently in the queue. Example: SIZE
Complexity: O(1)

PRINT Prints the entire queue from front to back. Example: PRINT Output
example: [Amir, Aliya, Dana] Complexity: O(n)

TICK Increases the logical time of the system. Example: TICK 7
Complexity: O(1)

STATS Shows service statistics: - how many people were served - average
waiting time Formula: averageWait = totalWait / servedCount Example:
STATS Complexity: O(1)

EXIT Stops the program. Example: EXIT

Short Defense Explanation: This program simulates a cafeteria queue.
Students can arrive, VIP students can join the front, and the system
serves them in order. The program also tracks waiting time and
calculates statistics. It uses ArrayDeque for the queue because it
allows O(1) insertion and removal at both ends. It also uses HashMap to
store arrival times with O(1) average lookup time.
